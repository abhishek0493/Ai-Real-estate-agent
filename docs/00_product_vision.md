# Project Name

AI Real Estate Sales Agent (Multi-Tenant SaaS)

# Purpose

Build a B2B SaaS product that provides an embeddable AI chatbot for real estate companies to automate lead qualification, property matching, and early-stage negotiation.

# Problem Statement

Real estate websites require users to manually browse listings and apply filters. This leads to:

- Low conversion rates
- Poor lead qualification
- Manual follow-ups
- Unstructured user data

We aim to solve this by creating an AI-driven conversational sales agent that:

1. Collects buyer requirements conversationally
2. Matches properties dynamically
3. Negotiates budget range
4. Qualifies lead intent
5. Stores lead data
6. Sends notification email
7. Marks lead status (interested / not interested)

# Target Users

Primary: Real Estate Businesses (B2B SaaS clients)
Secondary: End Users looking to buy property

# Core Value Proposition

- Higher lead conversion
- Structured lead qualification
- Reduced manual filtering
- Easy integration via SDK

# Positioning

Backend-centric SaaS system with AI orchestration.

# Objective

Build a production-grade multi-tenant SaaS backend that:

- Uses LLM function calling
- Maintains deterministic state transitions
- Implements clean architecture
- Deploys on Amazon ECS
- Uses PostgreSQL (RDS)
- Is RAG-extensible

# Success Criteria

- Clean modular architecture
- Strong separation of concerns
- 80% test coverage
- Dockerized
- CI/CD pipeline
- ECS deployment configuration
